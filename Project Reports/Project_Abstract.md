# Project Abstract

**Project Title:** AI-Powered Sanskrit NLP Framework for Morphological and Semantic Analysis  
**Course:** IT Project Management (21MGH303P)  
**Semester:** B.Tech. III Year / 6th Semester  
**Institution:** SRM Institute of Science and Technology, Delhi-NCR Campus, Modinagar, U.P.

---

This report presents the comprehensive IT Project Management practical work carried out for the project titled **“AI-Powered Sanskrit NLP Framework for Morphological and Semantic Analysis.”** The objective of this project is to design and plan an artificial intelligence-driven Natural Language Processing (NLP) framework capable of performing morphological analysis, semantic parsing, and structured knowledge extraction from classical Sanskrit texts. The framework is inspired by the computational modeling of **Pāṇini’s Ashtadhyayi grammar**.

The project is structured into five units as per the academic curriculum:

---

## Unit 1: Project Initiation and Planning
This unit focuses on requirement gathering through stakeholder interviews and document analysis. It includes:
- Problem identification and opportunity analysis  
- Feasibility study (technical, economic, and operational)  
- Project justification  
- Stakeholder analysis using the Power-Interest Grid  

---

## Unit 2: Cost Estimation and Financial Analysis
This unit covers multiple software cost estimation techniques:
- COCOMO (Intermediate Organic Mode)  
- Function Point Analysis (FPA)  
- Expert Judgment  

The estimated project cost ranges between **INR 33–41 Lakhs**. Financial feasibility is evaluated using:
- Net Present Value (NPV) — positive outcome  
- Payback Period — estimated at **3–4 years**  

Market demand analysis identifies key target segments such as universities, research institutions, digital libraries, and government cultural organizations, along with a phased demand strategy.

---

## Unit 3: Scheduling and Resource Management
Project scheduling is performed using:
- **PERT (Project Evaluation and Review Technique)** for expected time and variance analysis  
- **CPM (Critical Path Method)** to identify the critical path:  
  **A–B–D–E–G–H–I**, with a total duration of **218 days**

Additional analyses include:
- Resource levelling to address constraints  
- Project crashing to optimize time-cost trade-offs  

---

## Unit 4: Performance and Risk Management
Project performance is evaluated using **Earned Value Analysis (EVA)** with key metrics:
- Schedule Performance Index (SPI)  
- Cost Performance Index (CPI)  
- Cost Variance (CV)  
- Schedule Variance (SV)  

Risk management includes:
- Identification of key risks  
- Quantitative risk scoring  
- Mitigation strategies  

An **RMMM (Risk Mitigation, Monitoring, and Management)** plan is developed. Configuration management is implemented using **GitHub with Git Flow**. Unit testing includes **85 test cases**, achieving an **84.7% pass rate**.

---

## Unit 5: Audit, Agile, and DevOps Practices
This unit includes:
- Project audits (technical, financial, security, process, and quality)  
- Agile implementation using **Scrum** (sprint planning, stand-ups, reviews, retrospectives)  
- Integration of **Extreme Programming (XP)** practices such as:
  - Pair programming  
  - Test-driven development  
  - Continuous integration  
  - Refactoring  

Lean principles are applied to optimize workflow. DevOps practices are implemented using **Docker containerization**, including Dockerfiles, docker-compose, and CI/CD pipelines.

---

## Key Project Metrics

- **Total Estimated Cost:** INR 33–41 Lakhs  
- **Project Duration (Baseline):** 218 days (~10.5 months)  
- **Payback Period:** 3–4 years  
- **Net Present Value (10% Discount Rate):** Positive (~INR 12.6 Lakhs)
